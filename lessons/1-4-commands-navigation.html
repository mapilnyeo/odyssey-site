Commands & Navigation

Gemini CLI를 슈퍼파워처럼 쓰게 해주는 도구들입니다. 슬래시 명령어, 커스텀 TOML 명령, 키보드 단축키, 승인 모드, Chain-of-Thought 프롬프팅, Policy Engine까지 한 번에 마스터합니다.

터미널에서 시작:
gemini> /start-1-4

필수 슬래시 명령어

명령어 하는 일
/help 모든 명령어 목록 보기
/stats 현재 모델, 토큰 사용량 확인
/memory show 로드된 메모리 컨텍스트 보기
/memory add 새 메모리를 GEMINI.md에 추가
/memory refresh GEMINI.md 수정 후 다시 로드
/model 사용 모델 전환 (pro, flash 등)
/compress 컨텍스트가 길어질 때 압축
/clear 대화 초기화 후 새로 시작
/resume 이전 세션 복귀
/rewind 대화를 특정 지점으로 되돌리기
/plan 변경 없이 계획만 보기 모드로 전환
/commands 커스텀 명령어 포함 전체 목록

커스텀 명령어 (TOML)

.gemini/commands/ 폴더에 .toml 파일을 만들면 나만의 슬래시 명령어가 생깁니다.

# .gemini/commands/my-standup.toml
description = "오늘의 스탠드업 보고서 생성"

prompt = """
customer-data 폴더를 읽고 오늘 주목해야 할 사항을
스탠드업 형식(어제/오늘/블로커)으로 정리해줘.
"""
💡 파일 추가 후 /commands reload를 실행하면 재시작 없이 바로 사용 가능합니다.

키보드 단축키

⎋
Escape — 중단
Gemini가 작업 중일 때 누르면 즉시 멈춥니다. "잠깐, 잠깐만요" 버튼.
⎋⎋
Escape×2 — 되감기
대화 기록을 탐색하며 이전 지점으로 돌아갑니다. 파일 변경까지 함께 취소됩니다.
⇧⇥
Shift+Tab — 모드 전환
Default → Auto-edit → Plan 모드를 순환합니다.
↵
\ + Enter — 줄 바꿈
제출하지 않고 긴 프롬프트에서 새 줄로 이동합니다. Shift+Enter도 동일.

승인 모드 (Approval Modes)

Shift+Tab으로 3가지 모드를 순환합니다:

Default Mode
모든 액션에 승인 요청
가장 안전합니다. 파일 쓰기든 명령 실행이든 항상 먼저 물어봅니다. 처음 시작할 때, 또는 확신이 없을 때 사용하세요.
Auto-edit Mode
파일 수정은 자동, 명령 실행은 물어봄
반복적인 파일 편집 작업에서 속도를 높이고 싶을 때. Gemini를 어느 정도 신뢰할 때 사용합니다.
Plan Mode
읽기와 계획만, 절대 변경 없음
낯선 코드베이스를 탐색하거나 계획만 검토할 때. 실수로 뭔가가 변경되는 걱정 없이 마음껏 탐색합니다.

Policy Engine — AI 행동 규칙 설정

~/.gemini/policies/ 폴더에 .toml 파일을 만들어 Gemini CLI가 자동으로 할 수 있는 것과 항상 물어봐야 하는 것을 세밀하게 제어합니다.

# ~/.gemini/policies/my-rules.toml

# ✅ git status는 항상 자동 허용
[[rule]]
toolName = "run_shell_command"
commandPrefix = "git status"
decision = "allow"
priority = 100

# ❓ git push는 반드시 물어보기
[[rule]]
toolName = "run_shell_command"
commandPrefix = "git push"
decision = "ask_user"
priority = 200
🛡️ 3가지 decision 값:
allow — 자동 실행 / deny — 완전 차단 / ask_user — 항상 물어보기
공식 문서: geminicli.com/docs/reference/policy-engine

Chain-of-Thought 프롬프팅

Gemini CLI에는 "think" 같은 특수 키워드가 없습니다. 대신 직접 요청하거나 GEMINI.md에 내장합니다:

# 방법 1: 직접 요청
gemini> "단계별로 생각해보고 답해줘: 로열티 프로그램 참여율이 낮은 근본 원인이 뭐야?"

# 방법 2: Plan 모드에서 분석
gemini> /plan "고객 데이터 분석 계획 잡아줘"

Working with Files

Gemini CLI의 @ 심볼은 파일, 폴더, 이미지, URL의 내용을 직접 대화에 포함시키는 슈퍼파워입니다. 5가지 핵심 시나리오를 통해 실무에서 가장 자주 쓰는 패턴을 익힙니다.

터미널에서 시작:
gemini> /start-1-3

이 레슨에서 배우는 것

  • @파일명으로 특정 파일 내용을 대화에 포함시킨다
  • @폴더명/으로 폴더 전체를 한번에 분석한다
  • 문서 템플릿을 적용해 일관된 형식의 결과물을 만든다
  • 이미지를 Gemini에게 붙여넣어 분석을 요청한다
  • URL을 제공하면 웹 페이지 내용을 즉시 리서치한다

5가지 핵심 시나리오

1️⃣
단일 파일 → 정보 추출
한 파일에서 구조화된 정보를 뽑아냅니다.
gemini> "@customer-data/feedback_raw.csv 에서 부정적 피드백 상위 5개를 알려줘"
2️⃣
폴더 전체 → 패턴 종합
여러 파일에서 공통 테마를 발견합니다.
gemini> "@competitor-research/ 폴더의 모든 파일을 읽고 공통 성공 요인을 찾아줘"
3️⃣
템플릿 적용 → 일관된 출력
원하는 형식의 출력물을 템플릿으로 제어합니다.
gemini> "@customer-data/feedback_raw.csv 를 @templates/report_template.md 형식으로 정리해줘"
4️⃣
이미지 → 시각적 분석
스크린샷, 차트, 사진을 Gemini에게 직접 보여줍니다.
gemini> "@assets/app_screenshot.png 이 앱의 UX 문제점을 찾아줘"
5️⃣
웹 URL → 실시간 리서치
URL을 주면 해당 페이지를 즉시 읽고 분석합니다.
gemini> "https://starbucks.com/rewards 를 읽고 우리 프로그램과 비교해줘"

언제 무엇을 쓸까

상황 사용 패턴
@파일 특정 문서에서 정보를 뽑을 때
@폴더/ 비슷한 여러 파일에서 패턴을 찾을 때
@파일 + @템플릿 일관된 형식의 보고서나 문서가 필요할 때
이미지 붙여넣기 화면 캡처, 차트, UI 피드백이 필요할 때
URL 제공 최신 웹 정보나 경쟁사 페이지를 분석할 때

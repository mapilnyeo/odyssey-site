GEMINI.md — 프로젝트 영구 메모리

새 세션을 시작할 때마다 모든 컨텍스트를 다시 설명해야 하는 문제를 해결합니다. GEMINI.md를 한 번 설정하면, Gemini CLI는 매 세션마다 자동으로 읽고 프로젝트를 기억합니다.

터미널에서 시작:
gemini> /start-1-7

이 레슨에서 배우는 것

  • GEMINI.md가 해결하는 "컨텍스트 재설명 문제"를 이해한다
  • 글로벌 → 프로젝트 → 서브폴더 3단계 계층 구조를 파악한다
  • Basecamp Coffee 프로젝트의 GEMINI.md를 직접 작성한다
  • /memory show, /memory add, /memory refresh 명령어를 사용한다
  • Chain-of-Thought 지침을 GEMINI.md에 내장하는 법을 익힌다

GEMINI.md 계층 구조

# 더 구체적인 파일이 더 일반적인 파일을 덮어씁니다

~/.gemini/GEMINI.md
  # 🌍 글로벌 — 나의 작업 스타일, 선호 형식, 모든 프로젝트에 공통
      예: "항상 한국어로 답해줘", "불렛포인트 선호"

~/projects/basecamp-coffee/GEMINI.md
  # 📂 프로젝트 — 이 프로젝트의 컨텍스트
      예: 회사 개요, 문제, 해결책, 이해관계자

~/projects/basecamp-coffee/analysis/GEMINI.md
  # 📁 서브폴더 — 분석 작업에만 적용되는 특수 규칙
      예: "데이터 분석 시 항상 표 형식 사용"

좋은 GEMINI.md 예시

# complete-course/GEMINI.md

# Basecamp Coffee 프로젝트 컨텍스트

## 회사 개요
Basecamp Coffee: 태평양 북서부 45개 지점, 2015년 설립.
핵심 가치: 지역 원두, 커뮤니티 공간, 아티스트 지원.

## 현재 문제
- Basecamp Rewards 로열티 프로그램이 기대 미달
- $40K 투자했지만 참여율 저조
- 3개월 내 결과 없으면 폐기

## 해결 방향
Coffee Personality Quiz로 프로그램에 개성 부여

## 이해관계자
- Dana (중간 관리자), 경영진, 매장 매니저들

## AI 작업 지침
- 분석 시 단계별로 생각한 후 결론을 제시할 것
- 숫자와 데이터가 있으면 항상 인용할 것
- 실행 가능한 다음 단계를 항상 제안할 것

/memory 명령어

명령어 용도
/memory show 현재 로드된 모든 GEMINI.md 컨텍스트 확인
/memory add "내용" 새로운 내용을 프로젝트 GEMINI.md에 추가
/memory refresh GEMINI.md를 직접 수정한 후 다시 로드
💡 GEMINI.md에 Chain-of-Thought 지침을 넣으면 특별히 요청하지 않아도 Gemini가 항상 깊이 생각하고 답합니다.

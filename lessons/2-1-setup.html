Setup — Vibe Coding 준비

Module 1에서는 파일과 함께 일했습니다. Module 2에서는 직접 만듭니다. PM↔엔지니어 마인드셋 전환과 실제 웹앱 개발의 핵심 루프를 소개합니다.

터미널에서 시작:
gemini> /start-2-1

두 가지 역할 사이를 오간다

🎯
PM (Product Manager) 모드
무엇을 만들지 결정합니다. 요구사항 정의, 우선순위 설정, 디자인 방향 결정. 이것이 당신의 역할.
⚙️
Engineer 모드
어떻게 만들지 실행합니다. 코드 작성, 파일 생성, 서버 실행. 이것은 Gemini의 역할.

핵심 개발 루프

1 · Plan
요구사항 명확히 정의
무엇을 만들지, 어떻게 보여야 하는지, 어떻게 동작해야 하는지. REQUIREMENTS.md 문서로 기록.
2 · Build
Gemini가 코드 작성
요구사항을 주면 Gemini가 Next.js 앱을 처음부터 만들어줍니다. 코드를 이해할 필요 없습니다.
3 · Iterate
보고 → 피드백 → 수정
브라우저에서 결과물을 보고, 스크린샷을 붙여넣으며, 원하는 변경사항을 말하면 됩니다.
4 · Save
GitHub에 코드 저장
만든 코드를 GitHub에 올려 클라우드 백업과 버전 관리를 설정합니다.
5 · Deploy
Vercel로 실제 배포
Vercel CLI 한 줄로 인터넷에 게시. 실제 URL을 친구에게 보낼 수 있습니다.

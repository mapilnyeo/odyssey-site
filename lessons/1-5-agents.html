Agents — 병렬 처리의 마법

한 번에 하나씩 처리하는 것과 동시에 처리하는 것의 차이. Gemini CLI의 서브에이전트는 여러 파일을 동시에 처리해서 시간과 노력을 줄여줍니다. 현재 실험적 기능입니다.

터미널에서 시작:
gemini> /start-1-5

이 레슨에서 배우는 것

  • 서브에이전트가 무엇이고 언제 필요한지 이해한다
  • 경쟁사 분석 파일 3개를 에이전트로 동시에 처리해본다
  • 순차 처리 vs. 병렬 처리의 실질적인 차이를 느낀다
  • 어떤 작업에 에이전트가 적합한지, 그렇지 않은지 구분한다

서브에이전트란?

Gemini CLI가 복잡한 작업을 처리할 때, 하나씩 순서대로 하는 대신 여러 작업을 동시에 처리할 수 있습니다. 마치 한 명이 3개 파일을 차례로 읽는 대신, 3명을 고용해서 동시에 읽게 하는 것과 같습니다.

⏱️
순차 처리 (Before)
파일 A 읽기 → 파일 B 읽기 → 파일 C 읽기
총 시간: 3×T
⚡
병렬 처리 (After)
파일 A, B, C 동시에 읽기
총 시간: ≈1×T

실제 예시: 경쟁사 분석

Basecamp Coffee의 경쟁사 3개를 동시에 분석합니다:

gemini> "competitor-research 폴더의 starbucks, dutch_bros, blue_bottle 분석 파일을 동시에 읽고, 각각의 로열티 프로그램 핵심 성공 요인을 정리해줘"
🧪 Gemini CLI가 자동으로 각 파일을 병렬로 읽습니다. 복잡한 설정이 필요 없습니다.

서브에이전트가 빛나는 순간

적합한 경우 적합하지 않은 경우
비슷한 파일 여러 개 분석 단일 복잡한 파일 분석
독립적인 여러 연구 작업 순서가 중요한 작업
경쟁사·고객·시장 동시 조사 이전 결과가 다음에 필요한 경우
대용량 데이터 처리 섬세한 판단이 필요한 단일 작업

내장 서브에이전트

Gemini CLI에는 기본 내장 서브에이전트가 있습니다:

🔎
Codebase Investigator
코드베이스 탐색 전문 에이전트. 코드에서 패턴과 문제를 찾습니다.
💻
CLI Help Agent
CLI 명령어와 사용법에 대해 전문적으로 답변합니다.
🤖
Generalist Agent
일반 목적의 에이전트. 다양한 작업을 처리합니다.

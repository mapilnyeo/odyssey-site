File Exploration & Visualization

상속받은 혼란스러운 파일 더미를 처음 마주하는 상황을 시뮬레이션합니다. Gemini CLI로 파일을 탐색하고, Cursor 에디터로 시각화하는 환경을 설정합니다.

터미널에서 시작:
gemini> /start-1-2

이 레슨에서 배우는 것

  • Cursor 에디터에서 코스 폴더를 열어 파일 트리를 시각화한다
  • Gemini CLI가 폴더 구조 전체를 한 번에 이해하는 방법을 직접 체험한다
  • 혼란스러운 파일 더미에서 패턴과 주요 파일을 식별한다
  • Gemini CLI와 코드 에디터를 나란히 띄워 함께 작업하는 워크플로우를 익힌다

상속받은 파일 구조

전임 매니저가 남긴 파일들은 이런 모습입니다:

# complete-course/ 폴더 구조
📁 company-context/
  📄 SCENARIO.md
  📄 PREVIOUS_MANAGER_NOTES.md
📁 customer-data/
  📄 feedback_raw.csv
  📄 survey_results.md
📁 competitor-research/
  📄 starbucks_analysis.md
  📄 dutch_bros_analysis.md
  📄 blue_bottle_analysis.md
📁 organized/  # ← 나중에 채워질 폴더
💡 Gemini CLI는 이 폴더 전체를 몇 초 만에 읽고 전체 그림을 그려드립니다. 200페이지 메뉴얼을 읽은 직원처럼요.

작업 환경 설정

Step 1
Cursor에서 폴더 열기
Cursor 에디터를 열고 complete-course 폴더를 드래그해서 놓거나 File → Open Folder를 사용합니다. 좌측에 파일 트리가 보입니다.
Step 2
터미널 반분할 배치
터미널 창(Gemini CLI 실행 중)과 Cursor 창을 화면 좌우에 나란히 배치합니다. Gemini가 파일을 수정할 때 실시간으로 변화를 볼 수 있습니다.
Step 3
레슨 시작
터미널에서 /start-1-2를 입력하면 Gemini가 폴더를 탐색하고 현황 브리핑을 시작합니다.

핵심 능력: 한 번에 전부 읽기

Gemini CLI의 강점은 여러 파일을 동시에 소화한다는 점입니다. 아래처럼 요청해보세요:

gemini> "company-context 폴더의 모든 파일을 읽고 상황을 알려줘"
